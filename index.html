Node.js

Node.js theory:

  1. NodeJS created for:
    1. To create servers
    2. To create demon applications (automatically start in OS)

    3. Develop CLI (console applications)

    4. To create Desktop app, Mobile app
  2. Native module system is - Common JS Modules
  3. Versions of npm dependencies:
    • f.e. "nodemon": "^2.0.9"
    • "^" - means install new minor and patch dependency (NOT major version) to last version, f.e. when we clone old rep. and would like to update package to last version
    • f.e. "nodemon": "~2.0.9"
    • "~" means- means update only patch version

    • We could install for our proj only production dependencies by command npm i --only=prod

  4. REST API - это набор рекомендаций (1. How to write addresses for requests. 2. How to write answers) для написания веб-сервера, который реализует CRUD операции:
    • REST API main rules
    • REST API additional rules
    • REST API main answers
  5. Colorize console messages:
    • how-to-change-node-jss-console-font-color
  6. DataBases:
    1. DB divides into 2 main groups (school db divide guide):
      1. SQL (relatively): MySQL, SQL-Lite, PostgreSQL
      2. noSQL: MongoDB, Redis
    2. MongoDB => create a new cluster:
      1. Cloud Provider & Region - live "AWS" and choose your nearest city
      2. Press "Create cluster"
      3. go "Database Access" & "ADD NEW DATABASE USER"
      4. go "Network Access" & check from what IP-addresses you have access (if we need to allow connection from any ip-addresses don't make any rules or add this rule)
      5. go to "Clusters" menu and press "COLLECTIONS"
      • Rule: collections names always noun in plural, f.e. users, products
      • Info: MongoDB NodeJS Driver (another name MongoClient) - package that helps to connect code of NodeJS with database, another way to connect with database is f.e. express lib. and mongoose lib
  7. Authentication/authorization:
    • bcrypt lib working with stable nodeJS version
    • bcryptjs lib working with current (unstable) nodeJS version

    • Token consist:

      1. headers - algorithm and way of token creation (by default using const headers = {"alg": "HS256", "typ": "JWT"})
      2. payload - part that is encrypted ("id" f.e.)
      3. SECRET_KEY - usual string with any text
        • f.e const token = 'qwertyuiop.asdfghjkl.zxcvbnm', where "qwertyuiop" - headers encrypted, "asdfghjkl" - payload encrypted, "zxcvbnm" - SECRET_KEY encrypted
        • method from jsonwebtoken decrypts token and pull decrypted payload
  8. Архитектура приложений MVC и REST:
    • Understanding MVC pattern in Nodejs (об архитектуре - модель, представление, контроллер - (хорошо и с примером кода, что и где должно лежать по папочкам))
    • на примере абстрактных функций

    • и на примере чистой ноды
